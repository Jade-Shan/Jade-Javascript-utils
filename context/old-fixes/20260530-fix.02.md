# 检查问题
## 检查报告

### 1. `basic.ts` — NumUtil.add/sub 精度方案对大精度数仍有风险

**问题**：`NumUtil.add` 和 `NumUtil.sub` 使用 `n * m` 转为整数后计算，但当两数的小数位数不同时，较短的那个数乘以 `m` 后仍会引入浮点误差。例如 `NumUtil.add(0.1, 0.2)` 内部是 `(0.1 * 10 + 0.2 * 10) / 10 = (1 + 2) / 10 = 0.3`，恰好正确；但 `0.1 * 100` 并不精确等于 `10`。

**严重性**：低（已有的测试用例全部通过，实际使用场景影响有限）

**原代码** (第 110–116 行)：
```typescript
static add(n1: number, n2: number): number {
    let r1 = this.getDecimalPlaces(n1);
    let r2 = this.getDecimalPlaces(n2);
    let m = Math.pow(10, Math.max(r1, r2));
    let value = (n1 * m + n2 * m) / m;
    return value;
}
```

**建议优化**：
```typescript
static add(n1: number, n2: number): number {
    let r1 = this.getDecimalPlaces(n1);
    let r2 = this.getDecimalPlaces(n2);
    let m = Math.pow(10, Math.max(r1, r2));
    let value = (Math.round(n1 * m) + Math.round(n2 * m)) / m;
    return value;
}
```
同理 `sub` 也应改为 `Math.round(n1 * m) - Math.round(n2 * m)`。

### 2. `basic.ts` — NumUtil.div 结果多了一次 `* Math.pow(10, m+1) / 10` 的冗余操作

**问题**：`div` 方法在最后做了 `* Math.pow(10, m + 1)` 然后 `/10`，等价于 `* Math.pow(10, m)`，写法较难读且容易引入浮点误差。

**严重性**：低

**原代码** (第 151–159 行)：
```typescript
static div(n1: number, n2: number): number {
    let t1 = this.getDecimalPlaces(n1);
    let t2 = this.getDecimalPlaces(n2);
    let m = t2 - t1;
    let r1 = Number(n1.toString().replace(".", ""));
    let r2 = Number(n2.toString().replace(".", ""));
    let value = (r1 / r2) * Math.pow(10, m + 1);
    return value / 10;
}
```

**建议优化**：
```typescript
static div(n1: number, n2: number): number {
    let t1 = this.getDecimalPlaces(n1);
    let t2 = this.getDecimalPlaces(n2);
    let m = t2 - t1;
    let r1 = Number(n1.toString().replace(".", ""));
    let r2 = Number(n2.toString().replace(".", ""));
    return (r1 / r2) * Math.pow(10, m);
}
```

### 3. `basic.ts` — `StrUtil.trimLeft` / `trimRight` 使用已废弃 API

**问题**：`String.prototype.trimLeft()` 和 `String.prototype.trimRight()` 是非标准的别名，现代 ECMAScript 标准方法为 `trimStart()` / `trimEnd()`。

**严重性**：低（功能正确，但 TypeScript strict 模式下会有 deprecation 警告）

**原代码** (第 214–225 行)：
```typescript
static trimLeft(s: string): string {
    return s.trimLeft();
}
static trimRight(s: string): string {
    return s.trimRight();
}
```

**建议优化**：
```typescript
static trimLeft(s: string): string {
    return s.trimStart();
}
static trimRight(s: string): string {
    return s.trimEnd();
}
```

### 4. `web.ts` — `checkImageFileSize` 中硬编码提示文案与参数不一致

**问题**：`alert("图片大于500K，请压缩后上传")` 中的 500K 是硬编码的字符串，但实际阈值来自参数 `imgMaxSize`，如果传入其他值，用户看到的提示会与实际限制不符。

**严重性**：中

**原代码** (第 364–400 行)：
```typescript
static checkImageFileSize(fileInput: HTMLInputElement, imgMaxSize: number): boolean {
    // ...
    if (fileInput.files[0].size > imgMaxSize) {
        alert("图片大于500K，请压缩后上传");
        return false;
    }
    // ...
}
```

**建议优化**：
```typescript
static checkImageFileSize(fileInput: HTMLInputElement, imgMaxSize: number): boolean {
    // ...
    if (fileInput.files[0].size > imgMaxSize) {
        alert(`图片大于${Math.round(imgMaxSize / 1024)}K，请压缩后上传`);
        return false;
    }
    // ...
}
```

### 6. `web.ts` — `navigateUrl` 中使用已废弃的 `document.createEvent`

**问题**：Safari 兼容分支中使用了 `document.createEvent('Event')` + `evt.initEvent()`，这两个 API 早已被标记为废弃。现代浏览器（包括 Safari）均支持 `el.click()`。

**严重性**：低（作为 fallback 路径，实际很少被执行）

**原代码** (第 213–231 行)：
```typescript
private static navigateUrl(url: string, target?: string): void {
    let el = document.createElement("a");
    document.body.appendChild(el);
    el.href = url;
    if (target) { el.target = target; }
    if (el.click) {
        el.click();
    } else {
        try {
            let evt = document.createEvent('Event');
            evt.initEvent('click', true, true);
            el.dispatchEvent(evt);
        } catch (e) { }
    }
}
```

**建议优化**：
```typescript
private static navigateUrl(url: string, target?: string): void {
    let el = document.createElement("a");
    document.body.appendChild(el);
    el.href = url;
    if (target) { el.target = target; }
    el.click();
    el.remove();
}
```

### 8. `web.ts` — `checkMobile_zh_CN` 正则表达式未覆盖新号段

**问题**：`/^1[3458][0-9]\d{8}$/` 仅覆盖 13x/14x/15x/18x 号段，未覆盖 16x/17x/19x 等现有号段。

**严重性**：中（会导致合法手机号验证失败）

**原代码** (第 341 行)：
```typescript
static checkMobile_zh_CN(phoneno: string): boolean {
    return /^1[3458][0-9]\d{8}$/.test(phoneno);
}
```

**建议优化**：
```typescript
static checkMobile_zh_CN(phoneno: string): boolean {
    return /^1[3-9]\d{9}$/.test(phoneno);
}
```
